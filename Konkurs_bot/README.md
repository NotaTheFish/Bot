# 🎉 Giveaway Bot — бот для розыгрышей в Telegram

Бот управляется одним админом через личку. Проверяет подписку участников,
ведёт систему страйков и банов, умеет редактировать уже опубликованные объявления.

---

## Установка

```bash
pip install -r requirements.txt
python bot.py
```

### Переменные окружения (.env)

```
BOT_TOKEN=токен_от_BotFather
ADMIN_ID=твой_числовой_ID_в_Telegram
DATABASE_URL=postgresql://user:pass@host:5432/dbname
```

Свой ID можно узнать у @userinfobot.
Таблицы создаются автоматически при первом старте (`init_db()`).

### Деплой на Railway

- **Root Directory:** `Konkurs_bot`
- **Build Command:** `pip install -r requirements.txt`
- **Start Command:** `python bot.py`
- `DATABASE_URL` копируется из вкладки Variables твоего PostgreSQL-сервиса.

## Права бота

Бот должен быть **администратором** в каждом канале/чате, на который проверяется
подписка. Без этого не работают ни проверка подписки, ни создание пригласительных ссылок.

---

## Использование

### Создание конкурса
**➕ Создать конкурс** → название → ключ → текст объявления → количество призовых
мест → призы → каналы (переслать сообщение или ввести ID) → `готово`.

### Запуск
- **В чате:** написать `!ключ!` — бот опубликует объявление.
- **В канале:** выбрать конкурс → **📢 Запустить в канале** → ID канала.

### Редактирование на ходу
**✏️ Редактировать** → текст / призы / каналы.
Изменения автоматически применяются ко всем уже опубликованным объявлениям
(бот помнит их в таблице `published_messages`).

### Завершение
- **✅ Завершить конкурс** — финальная проверка подписок, начисление страйков,
  затем взвешенный случайный выбор победителей.
- **❌ Отменить конкурс** — без победителей.
- **🗑 Удалить конкурс** — полное удаление из БД (CASCADE).

## Премиум-эмодзи

С Bot API 9.4 (09.02.2026) боты могут отправлять кастомные эмодзи в **личку,
группы и супергруппы**, если у **владельца бота** (аккаунт, создавший его в
BotFather) есть Telegram Premium. Каналы в этот список НЕ входят.

Бот хранит объявление, название и призы как `message.html_text`, поэтому премиум-эмодзи
и форматирование (жирный, курсив, спойлер) сохраняются. Для чатов, которые не
принимают кастомные эмодзи, есть автофоллбэк: бот переотправляет сообщение
с обычными эмодзи вместо падения (`strip_custom_emoji`).

## Система страйков

Страйки начисляются **только** при нажатии «Завершить конкурс». Кто вышел из
каналов и вернулся до завершения — страйк не получает.

| Страйков | Шанс на победу |
|---|---|
| 0 | 100% (вес 1.0) |
| 1 | −15% (вес 0.85) |
| 2 | −50% (вес 0.5) |
| 3 | бан, шанс 0 |

Страйк = дисквалификация из текущего конкурса + глобальный счётчик.
При 3 страйках — автобан на все конкурсы бота.

**Управление:**
- **⚖️ Страйки и баны** (главное меню) — список нарушителей, снять страйк / снять бан.
- **↩️ Вернуть выбывших** (меню конкурса) — вернуть участника в конкретный конкурс.
  Не снимает глобальный страйк — это отдельное действие.

## Фоновые задачи

`bot.py` раз в сутки перепроверяет подписки всех участников активных конкурсов
(`SUBSCRIPTION_CHECK_INTERVAL`). Отписавшимся и админу приходят уведомления.
Это мониторинг — страйки тут НЕ начисляются.
